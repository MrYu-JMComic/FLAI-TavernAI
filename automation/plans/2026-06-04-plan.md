# 任务规格：世界书系统 — 触发词匹配与上下文注入测试加固

## 1. 任务ID

`TASK-WB-001`

## 2. 任务标题

世界书系统：补充触发词匹配和上下文注入的单元/集成测试，修复发现的边界问题

## 3. 任务描述

世界书系统（worldBooks）已完整实现，包括：
- **后端**：`modules/worldBooks.js` 提供 CRUD、触发词匹配（`matchWorldBookEntries`）、上下文构建（`buildWorldBookContext`）、at_depth 注入、selective 过滤、概率激活、sticky/cooldown/delay 状态机
- **前端**：`WorldView.vue` 提供世界书编辑器，`CharacterFormView.vue` 支持角色关联世界书，`ChatView.vue` 集成了 lorebook 上下文注入
- **数据库**：`world_books`、`world_book_entries`、`character_world_books`、`world_book_entry_state` 四张表

当前缺失的是对核心匹配逻辑的**系统性测试覆盖**。`backend.test.js` 中世界书相关测试仅覆盖基本 CRUD，未覆盖：
- 触发词匹配（关键词命中、正则模式、selective 过滤）
- 上下文构建与注入顺序
- 概率激活、sticky/cooldown/delay 状态机
- 边界情况（空触发词、禁用条目、超长内容）

本任务目标：补充测试 → 发现并修复边界问题 → 确保世界书系统可靠运行。

## 选择理由

在四个高优先级 AI风月参考功能中选择"世界书系统"的理由：
1. **最复杂**：涉及触发词匹配、正则、selective 过滤、概率、状态机等多个子系统
2. **测试缺口最大**：当前仅覆盖 CRUD，核心匹配逻辑无测试
3. **风险最高**：匹配逻辑直接影响 AI 对话上下文质量，bug 会静默影响用户体验
4. **安全可迭代**：纯测试+小修，不涉及 UI 改动或数据库迁移
