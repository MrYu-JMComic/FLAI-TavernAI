## 验收标准

### 空状态
- 展示图标 + 说明文字 + 操作按钮（CTA）
- 按钮跳转到对应的创建流程
- 样式与现有 UI 风格一致（使用 `muted-text`、`ghost-button` 等已有 class）

### 加载态
- 使用统一的 loading 指示器（非各视图各自为政）
- 加载中禁用交互操作

### 错误态
- 展示错误信息 + 重试按钮
- 网络错误和业务错误区分展示
- 不泄露敏感信息

## 实现策略
1. 先检查各视图现有的 `loading`、`error` 状态变量和条件渲染
2. 在已有 `v-if="loading"` 分支旁添加 `v-else-if="error"` 和 `v-else-if="!items.length"` 分支
3. 样式复用现有 CSS class，保持一致性
4. 如果多个视图的空/错状态结构高度重复，抽取公共组件
5. 最后运行 `npm run build` 和 `node scripts/check-encoding.mjs` 验证
