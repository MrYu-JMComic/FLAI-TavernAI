## 二、迭代目标

**在一个迭代内完成以下两项交付物：**

1. **世界书导入/导出功能** — 支持 JSON 格式导入导出，兼容 SillyTavern lorebook 格式
2. **后端单元测试** — 覆盖世界书模块核心逻辑

选择导入/导出作为主功能的理由：
- 纯本地功能，不涉及外部集成
- 改动量适中（后端 2 个端点 + 前端 2 个按钮 + 格式转换逻辑）
- 用户体验提升最大：世界书是劳动密集型内容，分享和迁移是刚需
- 与 SillyTavern 生态兼容可降低用户迁移成本

- 响应格式：

```json
    "scan_depth": 4,
    "lorebook_context_percent": 25,
    "entries": [
      {
        "name": "...",
        "trigger_keys": "...",
        "content": "...",
        "position": "before_char",
        "enabled": true,
        "regex_mode": false,
        "always_active": false,
        "depth": 0,
        "selective": false,
        "selective_logic": 0,
        "keys_secondary": "",
        "probability": 100,
        "use_probability": false,
        "inclusion_group": "",
        "group_weight": 0,
        "role": 0,
        "sticky": null,
        "cooldown": null,
        "delay": null
      }

  }

```

#### 批量导出端点

```
GET /api/world-books/export-all
```

- 导出当前用户的所有世界书
- 返回数组格式

#### 导入端点

**文件：** `backend/src/routes/worldBooks.js`（新增路由）

```
POST /api/world-books/import
```

- 接受 JSON body
- 自动检测格式：FLAI 原生 / SillyTavern lorebook
- 创建新世界书 + 所有条目
- 返回创建的世界书对象

#### SillyTavern 格式兼容

**新建文件：** `backend/src/modules/worldBookImport.js`

SillyTavern lorebook JSON 格式特征：
```json
{
  "entries": {
    "0": {
      "uid": 0,
      "key": ["trigger1", "trigger2"],
      "keysecondary": [],
      "comment": "Entry Name",
      "content": "...",
      "constant": false,
      "vectorized": false,
      "selective": false,
      "selectiveLogic": 0,
      "addMemo": true,
      "order": 100,
      "position": 0,
      "disable": false,
      "excludeRecursion": false,
      "preventRecursion": false,
      "delayUntilRecursion": false,
      "probability": 100,
      "useProbability": true,
      "depth": 4,
      "group": "",
      "groupOverride": false,
      "groupWeight": 100,
      "scanDepth": null,
      "caseSensitive": null,
      "matchWholeWords": null,
      "automationId": "",
      "role": null,
      "sticky": null,
      "cooldown": null,
      "delay": null
    }
  },
  "name": "World Book Name",
  "description": ""
}
```

映射关系：

| SillyTavern 字段 | FLAI 字段 | 转换规则 |
|-------------------|-----------|----------|
| `key` (array) | `trigger_keys` (string) | `key.join(',')` |
| `comment` | `name` | 直接映射 |
| `content` | `content` | 直接映射 |
| `constant` | `always_active` | 布尔映射 |
| `position` (int) | `position` (enum) | 0→before_char, 1→after_char, 2→at_start, 3→at_depth |
| `disable` | `enabled` | 取反 |
| `order` | `order_index` | 直接映射 |
| `selectiveLogic` | `selective_logic` | 直接映射 |
| `depth` | `depth` | 直接映射 |
| `group` | `inclusion_group` | 直接映射 |
| `role` | `role` | null→0 |

#### 导出为 SillyTavern 格式

```
GET /api/world-books/:id/export?format=sillytavern
```

- 反向映射，生成 SillyTavern 兼容 JSON

## 七、风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| SillyTavern 格式变更导致映射失效 | 低 | 中 | 格式检测采用宽松匹配，未知字段忽略不报错 |
| 大世界书导出性能问题 | 低 | 低 | 单用户数据量有限（通常 < 100 条目），无需优化 |
| 导入恶意 JSON | 中 | 中 | Zod schema 严格校验字段长度和类型，sanitizeText 处理文本字段 |
| 测试覆盖不足遗漏边界情况 | 中 | 中 | 优先覆盖 matchWorldBookEntries 的核心路径 |
| 编码问题（中文内容） | 低 | 高 | 运行 check-encoding.mjs 验证，所有文件写入使用 UTF-8 |
