### 2.2 上下文注入流程（已实现）

```
用户发送消息
  ↓
conversations.js POST /:id/messages
  ↓
1. getRecentMessages() — 取最近 N 条消息
2. matchWorldBookEntries(db, characterId, recentTexts, { conversationId })
   → 收集角色关联的世界书（character_id 字段 + character_world_books 表）
   → 收集会话级 chatLorebook_id
   → 按 scan_depth 截取最近消息
   → Phase 0: Sticky 条目（仍在粘性窗口内）
   → Phase 1: always_active 条目
   → Phase 2: 触发词匹配（支持正则/字面/选择性过滤/概率）
   → Phase 2.5: 分组互斥（加权随机选一个）
   → Phase 3: 递归激活（内容中包含其他条目的触发词，最多 5 层）
   → Phase 4: 更新状态（sticky/cooldown/delay 状态机）
   → Phase 5: Token 预算截断
  ↓
3. buildWorldBookContext(entries) — 合并 at_start/before_char/after_char 条目内容
4. buildModelMessagesV2() — 将世界书上下文拼入系统提示词
5. injectAtDepthEntries(messages, entries) — 将 at_depth 条目插入消息数组
  ↓
发送到 AI Provider
```

## 5. 验收标准

### AC-1：触发匹配引擎（测试验证）

```
场景：always_active 条目
  Given 一个 enabled=1, always_active=1 的条目
  When 调用 matchWorldBookEntries
  Then 该条目始终出现在结果中

场景：触发词字面匹配
  Given 条目 triggerKeys="魔法,魔力,法术"
  When 扫描文本包含"他使用了魔法攻击"
  Then 该条目被激活

场景：正则触发
  Given 条目 regex_mode=1, trigger_keys="\\d{3,}"
  When 扫描文本包含"房间号 402"
  Then 该条目被激活

场景：选择性过滤（selective_logic=0，副关键词命中时激活）
  Given 条目 triggerKeys="王城", keysSecondary="骑士,国王", selective=1
  When 扫描文本包含"王城"但不包含"骑士"或"国王"
  Then 该条目不被激活

场景：递归激活
  Given 条目 A triggerKeys="魔法学院"，内容包含"霍格沃茨"
  Given 条目 B triggerKeys="霍格沃茨"
  When 扫描文本包含"魔法学院"
  Then A 和 B 都被激活

场景：Sticky
  Given 条目 sticky=3
  When 第 1 条消息触发该条目
  Then 第 2、3、4 条消息该条目仍然激活
  And 第 5 条消息不再自动激活

场景：Cooldown
  Given 条目 cooldown=2
  When 该条目在第 1 条消息激活，第 2 条消息未触发
  Then 第 3 条消息该条目不被重新激活
  And 第 4 条消息可以重新激活

场景：分组互斥
  Given 条目 A 和 B 同属 inclusion_group="天气"
  When 两个条目都被触发
  Then 只有一个被选中
```

### AC-2：导入/导出

```
场景：导出
  Given 用户拥有一个包含 5 个条目的世界书
  When 调用 GET /api/world-books/:id/export
  Then 返回 JSON 包含 name, description, scanDepth, lorebookContextPercent, entries[]
  And 每个条目包含所有字段

场景：导入
  Given 一个符合格式的 JSON 文件
  When 调用 POST /api/world-books/import
  Then 创建新世界书 + 所有条目
  And 返回 { book: {...}, imported: 5, failed: 0 }
```

### AC-3：搜索与批量操作

```
场景：列表搜索
  Given 世界书列表包含"魔法世界"和"科幻设定"
  When 搜索框输入"魔法"
  Then 只显示"魔法世界"

场景：条目搜索
  Given 世界书包含 20 个条目
  When 搜索"王城"
  Then 只显示名称/触发词/内容包含"王城"的条目

场景：批量禁用
  Given 选中 3 个条目
  When 点击"批量禁用"
  Then 3 个条目全部变为 enabled=false
```

### AC-4：复制

```
场景：复制世界书
  Given 用户拥有一个包含 5 个条目的世界书
  When 点击"复制"
  Then 创建名称为"原名 (副本)"的新世界书
  And 包含所有条目的副本

场景：复制条目
  Given 世界书详情页
  When 点击条目的"复制"按钮
  Then 创建相同内容的新条目，排在原条目之后
```

## 7. 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 触发匹配 edge case（空触发词、特殊字符、超长内容） | 中 | 条目不触发或误触发 | 测试覆盖边界条件 |
| 导入 JSON 格式不兼容外部工具 | 中 | 用户导入失败 | 支持 Tavern 原始格式 + FLAI 扩展格式，导入时归一化 |
| 复制大量条目性能 | 低 | 前端卡顿 | 限制单次 ≤100 条目 |
| 中文编码问题 | 低 | 乱码 | 导出/导入均 UTF-8 JSON，测试含中文 |
| 状态机（sticky/cooldown/delay）持久化 | 中 | 重启后状态丢失 | `world_book_entry_state` 表已持久化，`_messageCounter` 从 DB 恢复 |
