## 二、技术方案文档（已实现版本）

### 数据模型

```sql
-- 世界书主表
CREATE TABLE world_books (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  name TEXT NOT NULL,
  description TEXT NOT NULL DEFAULT '',
  character_id TEXT,                       -- 旧版单角色关联（保留兼容）
  scan_depth INTEGER NOT NULL DEFAULT 1,
  lorebook_context_percent INTEGER NOT NULL DEFAULT 25,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- 世界书条目表
CREATE TABLE world_book_entries (
  id TEXT PRIMARY KEY,
  world_book_id TEXT NOT NULL,
  user_id TEXT NOT NULL,
  name TEXT NOT NULL DEFAULT '',
  trigger_keys TEXT NOT NULL DEFAULT '',
  content TEXT NOT NULL DEFAULT '',
  position TEXT NOT NULL DEFAULT 'before_char',  -- at_start/before_char/after_char/at_depth
  order_index INTEGER NOT NULL DEFAULT 0,
  enabled INTEGER NOT NULL DEFAULT 1,
  -- 高级字段
  regex_mode INTEGER NOT NULL DEFAULT 0,
  always_active INTEGER NOT NULL DEFAULT 0,
  depth INTEGER NOT NULL DEFAULT 0,
  selective INTEGER NOT NULL DEFAULT 0,
  selective_logic INTEGER NOT NULL DEFAULT 0,    -- 0=OR, 1=NOT, 2=NOT ALL
  keys_secondary TEXT NOT NULL DEFAULT '',
  use_probability INTEGER NOT NULL DEFAULT 0,
  probability REAL NOT NULL DEFAULT 100,
  inclusion_group TEXT NOT NULL DEFAULT '',
  group_weight INTEGER NOT NULL DEFAULT 0,
  role INTEGER NOT NULL DEFAULT 0,               -- 0=system, 1=user, 2=assistant
  sticky INTEGER,
  cooldown INTEGER,
  delay INTEGER,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (world_book_id) REFERENCES world_books(id) ON DELETE CASCADE
);

-- 角色-世界书多对多关联
CREATE TABLE character_world_books (
  character_id TEXT NOT NULL,
  world_book_id TEXT NOT NULL,
  PRIMARY KEY (character_id, world_book_id),
  FOREIGN KEY (character_id) REFERENCES characters(id) ON DELETE CASCADE,
  FOREIGN KEY (world_book_id) REFERENCES world_books(id) ON DELETE CASCADE
);

-- 条目状态（sticky/cooldown/delay 追踪）
CREATE TABLE world_book_entry_state (
  entry_id TEXT NOT NULL,
  conversation_id TEXT NOT NULL,
  last_triggered INTEGER NOT NULL DEFAULT 0,
  sticky_remaining INTEGER NOT NULL DEFAULT 0,
  cooldown_remaining INTEGER NOT NULL DEFAULT 0,
  PRIMARY KEY (entry_id, conversation_id)
);
```

### API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/world-books` | 列表（含 entry_count） |
| POST | `/api/world-books` | 创建世界书 |
| GET | `/api/world-books/:id` | 详情（含 entries + linkedCharacters） |
| PUT | `/api/world-books/:id` | 更新世界书 |
| DELETE | `/api/world-books/:id` | 删除世界书 |
| POST | `/api/world-books/:id/entries` | 添加条目 |
| PUT | `/api/world-books/:id/entries/:entryId` | 更新条目 |
| DELETE | `/api/world-books/:id/entries/:entryId` | 删除条目 |
| GET | `/api/characters/:id/world-books` | 角色关联的世界书列表 |
| POST | `/api/characters/:id/world-books` | 关联世界书到角色 |
| DELETE | `/api/characters/:id/world-books/:bookId` | 取消关联 |

### 触发词匹配与上下文注入逻辑

```
用户发送消息
  │
  ├─ 1. 收集关联世界书：角色关联 + 聊天专属
  │
  ├─ 2. 对每个世界书，按 scanDepth 扫描最近 N 条消息
  │     ├─ alwaysActive 条目 → 直接匹配
  │     ├─ regexMode 条目 → 正则匹配
  │     └─ 普通条目 → 关键词匹配
  │           ├─ selective=true → 检查次级键 + 逻辑(OR/NOT/NOT ALL)
  │           └─ useProbability → 概率过滤
  │
  ├─ 3. 递归激活：已匹配条目触发其他条目
  │
  ├─ 4. 包含组去重：同组保留 groupWeight 最高的
  │
  ├─ 5. Token 预算裁剪：按 lorebookContextPercent 限制总注入量
  │
  ├─ 6. 按 position 注入：
  │     ├─ at_start → 系统提示最前
  │     ├─ before_char → 角色设定前
  │     ├─ after_char → 角色设定后
  │     └─ at_depth → 消息流中指定深度
  │
  └─ 7. 更新 sticky/cooldown/delay 状态
```
