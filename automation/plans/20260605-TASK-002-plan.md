## 二、详细描述

创建 `frontend/src/views/PresetView.vue` 预设管理页面，包含以下功能模块：

### 2.1 预设列表视图

- 展示所有预设，卡片式布局
- 每个预设卡片显示：名称、系统提示词摘要（前 80 字符）、关键参数摘要（temperature / max_tokens）、默认标记（⭐）
- 默认预设卡片高亮边框
- 列表顶部操作栏：「新建预设」按钮

### 2.2 预设编辑表单

点击卡片进入编辑模式（或新建时直接进入表单），包含以下字段：

| 字段 | 控件 | 范围 | 默认值 |
|------|------|------|--------|
| 名称 | `<input type="text">` | 最长 100 字符 | "未命名预设" |
| 系统提示词 | `<textarea>` | 最长 50000 字符 | 空 |
| Temperature | `<input type="range">` + 数字显示 | 0 ~ 2，步长 0.1 | 1.0 |
| Max Tokens | `<input type="number">` | 1 ~ 128000 | 4096 |
| Top P | `<input type="range">` + 数字显示 | 0 ~ 1，步长 0.05 | 1.0 |
| Frequency Penalty | `<input type="range">` + 数字显示 | -2 ~ 2，步长 0.1 | 0 |
| Presence Penalty | `<input type="range">` + 数字显示 | -2 ~ 2，步长 0.1 | 0 |
| 设为默认 | `<input type="checkbox">` | — | false |

### 2.3 操作按钮

- **保存**：新建时调用 `createPreset()`，编辑时调用 `updatePreset()`
- **取消**：返回列表视图
- **删除**：确认弹窗后调用 `deletePreset()`（不可删除最后一个预设的提示）
- **设为默认**：调用 `setDefaultPreset()`，列表即时刷新

### 2.4 路由集成

- App.vue 的 `views` 对象添加 `presets`
- `parseRoute()` 添加 `#/presets` 解析
- `routeToHash()` 添加 presets 映射

### 2.5 导航入口

- BaseLayout.vue 的 `<nav>` 中添加「预设」按钮
- 使用 `SlidersHorizontal` 图标（lucide-vue-next 已有依赖）
- 当 `currentRoute === 'presets'` 时高亮

## 六、风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 侧边栏布局改动影响其他页面 | 低 | 中 | 仅添加一个 `<button>`，不改现有结构 |
| slider 控件在移动端体验差 | 低 | 低 | 使用 `<input type="range">` 原生控件，移动端原生支持好 |
| 编码问题（中文提示词内容） | 低 | 低 | 文件使用 UTF-8 编码，运行 check-encoding 验证 |
| 后端 API 返回格式与前端不匹配 | 极低 | 低 | API 层已有封装，后端 toPreset() 已做 camelCase 映射 |
| 最后一个预设被删除 | 低 | 低 | 前端删除前检查列表长度，仅剩一个时提示不可删除 |

**影响范围：** 仅涉及预设相关模块，不影响角色卡、世界书、对话等核心功能。
