### 3.3 详细描述

#### 方向 A：后端单元测试（主交付物）

新建 `backend/src/tests/worldBooks.test.js`，使用内存数据库，覆盖以下测试组：

**A1. 世界书 CRUD**
- 创建世界书（含名称/描述/scanDepth/lorebookContextPercent）
- 获取世界书详情（含条目列表和 linkedCharacters）
- 更新世界书字段
- 删除世界书（级联删除条目）
- 名称校验（空名称、超长名称抛异常）

**A2. 条目 CRUD**
- 创建条目（默认值填充）
- 更新条目（字段部分更新）
- 删除条目
- 条目排序（order_index）
- 条目 toggle enabled/disabled

**A3. 角色 ↔ 世界书关联**
- linkWorldBookToCharacter / unlinkWorldBookFromCharacter
- listCharacterWorldBooks（含 order_index 排序）
- 同一世界书关联多个角色

**A4. 触发词匹配引擎（核心）**
- 字符串匹配（大小写不敏感）
- 正则匹配（regex_mode）
- 混合模式（`/regex/flags` 语法内嵌在 triggerKeys 中）
- always_active 条目无条件激活
- 无效正则跳过不崩溃

**A5. 高级匹配特性**
- Selective 过滤：副关键词命中/未命中/全命中三种逻辑
- 概率激活（use_probability + probability）
- 分组互斥（inclusion_group + groupWeight 加权随机）
- Sticky：激活后持续 N 条消息
- Cooldown：停用后冷却 N 条消息
- Delay：首次出现后延迟 N 条消息激活

**A6. 递归激活**
- 匹配条目内容中包含其他条目的触发词 → 递归展开
- 最大递归深度 5 层
- 无新匹配时终止

**A7. at_depth 注入**
- injectAtDepthEntries 在正确位置插入消息
- 深度排序（深的先插入）
- role 映射（0→system, 1→user, 2→assistant）

**A8. Token 预算截断**
- 匹配条目按 order_index 排序后截断到 lorebookContextPercent 预算

**A9. 边界场景**
- 空世界书（无条目）→ 返回空数组
- 禁用条目不参与匹配
- 超长内容截断（10000 字符限制）
- 不存在的世界书/条目返回 null

#### 方向 B：导入/导出

**后端新增端点：**

```
GET  /api/world-books/:id/export   → 导出单个世界书为 JSON
POST /api/world-books/import        → 从 JSON 导入创建世界书
```

导出格式：
```json
{
  "_flai_export_version": 1,
  "exported_at": "2026-06-06T...",
  "world_book": {
    "name": "...",
    "description": "...",
    "scan_depth": 4,
    "lorebook_context_percent": 25,
    "entries": [{ ... }]
  }
}
```

导入逻辑：
- 校验 JSON 结构（zod schema）
- 检测 SillyTavern 格式（`entries` 为对象而非数组）→ 自动转换
- 单次导入限制 100 条目
- 创建新世界书 + 批量插入条目

**前端改动：**
- 列表页添加"导入"按钮（文件选择 + JSON 解析）
- 详情页添加"导出"按钮（下载 JSON 文件）
