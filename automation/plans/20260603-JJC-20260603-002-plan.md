## 三、任务规格

### 任务 ID
`JJC-20260603-002`

### 任务描述

为已有的 Mod 系统后端实现完整的前端管理 UI，使用户可以在设置页面中创建、编辑、删除、排序 Mod，并控制启用/禁用状态。

### DoD（完成定义）

#### 功能完整性

- [ ] 1. **API 封装**：在 `frontend/src/api.js` 中添加 Mod 相关 API 函数：
  - `fetchMods()` — GET `/api/mods`
  - `createMod(payload)` — POST `/api/mods`
  - `updateMod(id, payload)` — PUT `/api/mods/:id`
  - `deleteMod(id)` — DELETE `/api/mods/:id`
  - `reorderMods(orderedIds)` — PUT `/api/mods/order`

- [ ] 2. **Mod 管理面板**：在 `SettingsView.vue` 中新增 Mod 管理区域（或创建独立组件 `ModManager.vue` 后嵌入），包含：
  - Mod 列表（卡片/行），显示名称、类型、启用状态
  - 创建 Mod 对话框/表单，含字段：name、description、type（下拉：提示词注入/文风增强/自定义）、content（多行文本）、enabled（开关）
  - 编辑 Mod 对话框/表单（复用创建表单）
  - 删除 Mod（含确认对话框）
  - 启用/禁用切换开关
  - 拖拽排序（或上下移动按钮）

- [ ] 3. **三种 Mod 类型**的表单应有差异化的提示文本：
  - `prompt_inject`（提示词注入）：说明内容将直接追加到系统提示词
  - `style_enhance`（文风增强）：说明内容将以「[文风要求]」前缀注入
  - `custom`（自定义）：说明内容将以「[Mod: 名称]」前缀注入

- [ ] 4. **导航入口**：在 SettingsView 的侧边栏/标签中添加「Mod 管理」入口

#### 质量保证

- [ ] 5. 后端现有测试全部通过（`npm test`）
- [ ] 6. 新增至少 3 个后端测试覆盖：
  - 创建 Mod 后 `getEnabledModsForUser` 返回该 Mod
  - `buildModSystemPrompt` 正确拼接三种类型 Mod 内容
  - `reorderMods` 正确更新 order_index
- [ ] 7. 前端构建通过（`npm run build`）
- [ ] 8. `node scripts/check-encoding.mjs` 通过
- [ ] 9. 所有新写/修改的文件为 UTF-8 编码

#### 文档

- [ ] 10. 写入实施报告到 `automation/reports/`

## 五、技术方案概要

### 前端实现

#### Step 1：API 封装

**文件**：`frontend/src/api.js`

```javascript
// Mod 管理
export async function fetchMods() { /* GET /api/mods */ }
export async function createMod(payload) { /* POST /api/mods */ }
export async function updateMod(id, payload) { /* PUT /api/mods/:id */ }
export async function deleteMod(id) { /* DELETE /api/mods/:id */ }
export async function reorderMods(orderedIds) { /* PUT /api/mods/order */ }
```

#### Step 2：ModManager 组件

**文件**：`frontend/src/components/ModManager.vue`

组件结构：
```
┌─────────────────────────────────────────┐
│ Mod 管理                    [+ 新建 Mod] │
├─────────────────────────────────────────┤
│ ┌─ Mod 卡片 ──────────────────────┐     │
│ │ 📝 提示词注入 │ 名称 │ [开] [编辑] [删] │
│ │ 内容预览（截断 100 字）           │     │
│ └──────────────────────────────────┘     │
│ ┌─ Mod 卡片 ──────────────────────┐     │
│ │ 🎨 文风增强  │ 名称 │ [开] [编辑] [删] │
│ │ 内容预览                        │     │
│ └──────────────────────────────────┘     │
│                                          │
│ 拖拽排序提示                             │
└─────────────────────────────────────────┘
```

#### Step 3：集成到 SettingsView

**文件**：`frontend/src/views/SettingsView.vue`

在现有的设置标签/区域中添加 Mod 管理入口，引入 `ModManager` 组件。

#### Step 4：后端测试

**文件**：`backend/src/tests/mods.test.js`（新增）

测试用例：
1. 创建 Mod 并验证 `getEnabledModsForUser` 返回
2. `buildModSystemPrompt` 对三种类型 Mod 的输出格式
3. `reorderMods` 正确更新排序

### 数据流

```
用户操作 → ModManager.vue → api.js → /api/mods → mods.js → SQLite mods 表
                                                    ↓
                                              getEnabledModsForUser()
                                                    ↓
                                          buildModSystemPrompt() → 聊天时注入系统提示词
```
