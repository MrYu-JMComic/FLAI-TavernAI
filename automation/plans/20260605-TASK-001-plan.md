## 一、现状分析

经过逐文件审查，世界书系统**核心功能已完成**。以下为已实现的功能清单：

### ✅ 后端（完整）

| 模块 | 文件 | 功能 |
|------|------|------|
| 数据库 | `backend/src/db.js` | `world_books`、`world_book_entries`、`character_world_books`、`world_book_entry_state` 四张表 |
| CRUD | `backend/src/modules/worldBooks.js` | 世界书和条目的完整 CRUD，含递归激活、selective 过滤、概率触发、分组互斥、sticky/cooldown/delay 状态追踪 |
| 路由 | `backend/src/routes/worldBooks.js` | REST API（GET/POST/PUT/DELETE），含 AI 草稿助手端点 |
| AI 助手 | `backend/src/services/worldBookAssistant.js` | 基于 tool completion 的世界书自动生成，支持流式输出 |
| 校验 | `backend/src/validations/schemas.js` | Zod schema 校验世界书和条目的所有字段 |

### ✅ 触发注入逻辑（完整）

- **触发词匹配：** 支持普通文本匹配和正则模式（`regexMode`）
- **递归激活：** 条目内容中出现其他条目的触发词时递归激活，最大深度 5
- **Selective 过滤：** 主关键词 + 副关键词组合过滤，支持 3 种逻辑
- **概率激活：** 条目按百分比概率激活
- **互斥分组：** 同一 `inclusion_group` 中仅保留一个条目（按权重随机）
- **Sticky / Cooldown / Delay：** 完整的消息级状态追踪
- **at_depth 注入：** 指定深度插入消息数组，支持 system/user/assistant 角色
- **Token 预算：** 按 `lorebookContextPercent` 截断，按 `order_index` 排序
- **扫描深度：** `scan_depth` 控制扫描最近 N 条消息

### ✅ 前端（完整）

| 模块 | 文件 | 功能 |
|------|------|------|
| 世界书管理 | `frontend/src/views/WorldBookView.vue` | 完整的列表/详情/编辑 UI，含 AI 助手面板 |
| 角色卡关联 | `frontend/src/views/CharacterFormView.vue` | 角色表单中关联世界书选择器，支持多选 |
| 聊天设置 | `frontend/src/components/chat/ChatSettingsDrawer.vue` | 对话级世界书绑定（`chatLorebookId`） |
| API 层 | `frontend/src/api.js` | 完整的世界书 REST API 封装 |

### ✅ 聊天集成（完整）

在 `backend/src/routes/conversations.js` 的消息处理流程中：
1. `matchWorldBookEntries()` 根据角色 ID 和最近消息匹配条目
2. `buildWorldBookContext()` 构建注入内容
3. `injectAtDepthEntries()` 在消息数组中插入 at_depth 条目
4. 注入到系统提示词（`[世界书补充信息]` 区块）

### ✅ 角色导入/导出（完整）

`routes/characters.js` 的 export 端点已包含世界书数据，import 端点可创建关联世界书和条目。

**涉及文件：**
- `backend/src/routes/worldBooks.js` — 新增 export/import 端点
- `frontend/src/views/WorldBookView.vue` — 新增导入/导出按钮
- `frontend/src/api.js` — 新增 API 调用

**实现步骤：**

**DoD：**
- [ ] 可导出单个世界书为 JSON 文件（含所有条目完整数据）
- [ ] 可从 JSON 文件导入创建新世界书

**验收标准：**

2. 点击"导入"选择 JSON 文件，成功创建世界书和条目

**DoD：**
- [ ] 中文内容的 token 估算更接近实际消耗
- [ ] 不影响现有匹配逻辑
