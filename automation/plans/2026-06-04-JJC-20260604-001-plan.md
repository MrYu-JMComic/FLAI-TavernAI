### 4.2 Phase 2：世界书导入/导出

#### 4.2.1 后端

**新建文件：** `backend/src/modules/worldBookImport.js`

负责格式转换逻辑，将 SillyTavern lorebook JSON 映射到 FLAI schema。

**SillyTavern 格式特征：**
```json
{
  "entries": {
    "0": {
      "uid": 0,
      "key": ["trigger1", "trigger2"],
      "keysecondary": [],
      "comment": "Entry Name",
      "content": "...",
      "constant": false,
      "selective": false,
      "selectiveLogic": 0,
      "addMemo": true,
      "order": 100,
      "position": 0,
      "disable": false,
      "probability": 100,
      "useProbability": true,
      "depth": 4,
      "group": "",
      "groupWeight": 100,
      "role": null,
      "sticky": null,
      "cooldown": null,
      "delay": null
    }
  },
  "name": "World Book Name",
  "description": ""
}
```

**字段映射表：**

| SillyTavern 字段 | FLAI 字段 | 转换规则 |
|-------------------|-----------|----------|
| `key` (array) | `triggerKeys` (string) | `key.join(',')` |
| `comment` | `name` | 直接映射 |
| `content` | `content` | 直接映射 |
| `constant` | `alwaysActive` | 布尔映射 |
| `position` (int) | `position` (enum) | 0→before_char, 1→after_char, 2→at_start, 3→at_depth |
| `disable` | `enabled` | 取反 |
| `order` | `orderIndex` | 直接映射 |
| `selectiveLogic` | `selectiveLogic` | 直接映射 |
| `depth` | `depth` | 直接映射 |
| `group` | `inclusionGroup` | 直接映射 |
| `role` | `role` | null→0 |
| `sticky` | `sticky` | 直接映射 |
| `cooldown` | `cooldown` | 直接映射 |
| `delay` | `delay` | 直接映射 |
| `probability` | `probability` | 直接映射 |
| `useProbability` | `useProbability` | 直接映射 |

**修改文件：** `backend/src/routes/worldBooks.js`

## 6. 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| SillyTavern 格式版本差异（v1 vs v2） | 中 | 导入失败 | 格式检测采用宽松匹配，未知字段静默忽略不报错 |
| 递归激活测试中的随机性 | 低 | 测试不稳定 | 互斥分组测试用 mock `Math.random` 或固定 seed |
| 导入大量条目导致性能问题 | 低 | 响应慢 | 限制单次导入最多 100 条目 |
| 上下文预算估算不准确 | 中 | 用户误导 | 明确标注"估算"，使用 `~1200 tokens` 格式 |
| 流式响应中 worldBookMeta 丢失 | 低 | UI 不显示 | 在 `done` 事件中始终包含，即使为空对象 |
| 导入恶意 JSON | 中 | 中 | Zod schema 严格校验字段长度和类型，`sanitizeText` 处理文本字段 |
| 编码问题（中文内容） | 低 | 高 | 运行 `check-encoding.mjs` 验证，所有文件写入使用 UTF-8 |
