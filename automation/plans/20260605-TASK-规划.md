# 任务规划：统一 Vue 视图的空状态、加载态和错误态

## 任务ID
TASK-20260605-001

## 任务描述

### 背景
FLAI-TavernAI 前端各 Vue 视图的空状态、加载态和错误态处理方式不一致：
- **HomeView** 有骨架屏（skeleton）+ 错误面板，是最佳实践
- **PresetView** 有 loading-spinner + error-state + empty-state，较完整
- **WorldBookView** 类似 PresetView 的模式
- **ChatView** 仅显示 "正在加载对话..." 纯文本
- **CharacterFormView** 仅显示 "正在加载角色..." 纯文本
- **SettingsView** 无统一加载/错误态
- **NpcPanel** 显示 "加载中..." 纯文本
- **SaveLoadPanel** 显示 "正在加载存档..." 纯文本
- **CharacterImagePanel** 显示 "正在加载立绘..." 纯文本
- **EconomyPanel** 有自定义 spinner 但样式不统一
- **TalentRollDialog** 有 muted-text loading + empty-state

### 目标
1. 提取可复用的 UI 状态组件（Skeleton、EmptyState、ErrorState）
2. 统一所有视图的状态处理模式
3. 保持现有功能不变，仅改善视觉呈现和一致性

### 不在范围内
- 不改变业务逻辑
- 不新增功能
- 不修改后端代码
- 不改变现有交互流程

## 验收标准

- [ ] **加载态**：所有数据加载场景显示 skeleton 或 spinner，而非纯文本
- [ ] **空状态**：所有列表/数据为空时显示引导性空状态面板，包含图标和操作入口
- [ ] **错误态**：所有请求失败场景显示错误消息+重试按钮，不再静默失败
- [ ] **一致性**：三个基础组件在所有视图中复用，样式统一
- [ ] **无回归**：现有功能（创建、编辑、删除、搜索、筛选等）不受影响
- [ ] **构建通过**：`cd frontend && npm run build` 零错误
- [ ] **编码合规**：所有新文件为 UTF-8 编码
