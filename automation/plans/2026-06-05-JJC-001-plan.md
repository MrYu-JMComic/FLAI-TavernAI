## DoD（完成定义）
1. ChatView 加载对话时显示骨架屏或 spinner，而非仅文字
2. ChatView 空对话（无消息）时显示引导提示
3. ChatView 对话加载失败时显示可操作的错误面板（含重试按钮）
4. SettingsView 页面加载时显示加载指示器
5. SettingsView 配置加载失败时显示错误面板
6. CharacterFormView 加载/保存失败时有视觉反馈
7. 所有新增状态样式复用已有 CSS 类（`empty-state`、`loading-state`、`error-state`、`loading-spinner`）
8. 不引入新依赖，不改变现有交互逻辑

## 验收标准
1. 在网络断开时打开 ChatView，能看到错误面板和重试按钮，而非空白页面
2. 新建对话（无消息）时，ChatView 显示友好的空状态引导
3. SettingsView 首次加载时有明显的加载指示
4. SettingsView 加载失败时有错误面板（非仅 toast）
5. CharacterFormView 编辑不存在的角色 ID 时，显示错误而非白屏
6. 所有状态切换无闪烁（loading → content/error 过渡自然）
7. `node scripts/check-encoding.mjs` 通过
8. 现有测试不受影响
