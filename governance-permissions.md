## 禁止操作（需皇上亲自批准）

- 删除文件
- 修改 `.env` 或配置文件
- 修改 `governance.md` 或 `AGENTS.md`
- 推送到 Git 远程
- 安装/卸载 npm 依赖
- 修改数据库 schema

## 越权上报机制

当 agent 发现任务超出自身权限时：

1. **立即停止** — 不执行越权操作
2. **记录越权事件** — 写入 `automation/reports/escalation-YYYY-MM-DD.md`
3. **向上汇报** — 向调用方返回越权报告
4. **⚡ 乐自动重派** — 收到越权上报后，乐立即重新派发给有权限的部门，不需要皇上再次下旨，格式如下：

```
⚠️ 越权上报
- 请求操作: [具体操作]
- 当前角色: [agent id]
- 所需权限: [哪个等级才有此权限]
- 建议: [应由哪个角色执行 / 需皇上批准]
```

## 门下省审核规则

所有代码变更必须经过门下省审核：

1. 检查变更是否在执行者权限范围内
2. 检查变更是否破坏现有功能（运行测试）
3. 检查是否涉及禁止操作
4. 审核通过 → 允许合并
5. 审核不通过 → 封驳，打回尚书省重新派发
